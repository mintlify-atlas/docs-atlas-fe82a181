The Dashboard module exposes a single aggregation endpoint that gives every authenticated user an instant snapshot of store health. The response is computed in real time from live database queries — sales totals, product counts, low-stock alerts, active client count, monthly sales trends, product distribution by category, and pending purchase orders are all returned in one call. No date range parameters are required; the service uses `LocalDateTime.now()` to scope "today" and "this year" queries automatically.

---

## GET /api/dashboard/stats

Returns a map of real-time business KPIs for the current store state.

**`GET http://localhost:8080/api/dashboard/stats`**

### Authorization

<ParamField header="Authorization" type="string" required>
  Bearer token — accessible by **all** authenticated roles (`ADMIN`, `VENDEDOR`, `ALMACENERO`).
</ParamField>

### Response `200 OK`

<ResponseField name="ventasHoy" type="BigDecimal">
  Total monetary amount (sum of `total`) for all sales registered today.
</ResponseField>

<ResponseField name="totalVentas" type="Long">
  Cumulative count of all sales ever recorded in the system (including voided).
</ResponseField>

<ResponseField name="totalProductos" type="Long">
  Total number of active products registered in the catalog.
</ResponseField>

<ResponseField name="totalClientes" type="Long">
  Total number of active clients registered in the system.
</ResponseField>

<ResponseField name="comprasPendientes" type="Integer">
  Count of purchase orders currently in `PENDIENTE` state — awaiting stock receipt.
</ResponseField>

<ResponseField name="productosStockBajo" type="array">
  List of products whose `stockActual` is at or below their `stockMinimo`. Each entry contains:

  <Expandable title="productosStockBajo[] fields">
    <ResponseField name="id" type="Long">
      Product ID.
    </ResponseField>
    <ResponseField name="nombre" type="string">
      Product name.
    </ResponseField>
    <ResponseField name="stockActual" type="Integer">
      Current stock level.
    </ResponseField>
    <ResponseField name="stockMinimo" type="Integer">
      Configured minimum stock threshold.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="alertasStock" type="Integer">
  Numeric count of products with stock below minimum — equivalent to `productosStockBajo.length`. Useful for badge counters in a UI.
</ResponseField>

<ResponseField name="ventasPorMes" type="object">
  Monthly sales totals for the **current year**, keyed by month number (1–12). Months with no sales are included with a value of `0`. Useful for rendering bar charts or trend lines.
</ResponseField>

<ResponseField name="productosPorCategoria" type="array">
  Product count grouped by category. Each entry contains:

  <Expandable title="productosPorCategoria[] fields">
    <ResponseField name="categoria" type="string">
      Category name.
    </ResponseField>
    <ResponseField name="cantidad" type="Long">
      Number of products in this category.
    </ResponseField>
  </Expandable>
</ResponseField>

### Error Codes

| Status | Meaning |
|--------|---------|
| `401 Unauthorized` | Token missing, invalid, or expired. |
| `403 Forbidden` | Request reached the endpoint without a valid authenticated principal. |

<Tip>
  Poll this endpoint on a short interval (e.g. every 30 seconds) from your frontend to keep KPI cards and charts up to date without requiring a full page reload. The entire response is computed from live queries — there is no caching layer.
</Tip>
